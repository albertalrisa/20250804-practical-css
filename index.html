Padding/Margin Shorthand
				
					.block {
						padding-inline: 2rem;
						margin-block: 1rem;
					}
Previously, if we want to set left/right or top/bottom with the same value, we ended up defining something like `margin: 0 auto`, where 0 means top/bottom margin is `0`, and left/right margin is `auto`.
This means even if we only want to define the left/right margin or padding, we'll ended up with having to define both then overriding the value. The shorthand syntax also makes it impossible to define the same top/bottom value but different left/right value.
This is now solved with the new `margin-inline`, `margin-block`, `padding-inline`, and `padding-block` property. These new properties neatly follow the text direction of the page. The `-inline` property means were defining in the direction of the text. In a left-to-right text like English, this is the left/right value. The `-block` property is defines the cross axis, which means top/bottom for left-to-right text.

Column and Row Size
Flex (`fr`) allows us to define a flexible length in a grid container. The `fr` unit represents a fraction of the leftover space. For example, a `3fr` div and `2fr` div placed in a 800px container will split the container to `3/5` (480px) and `2/5` (320px).
`minmax` allows us to specify the minimum and maximum size of a cell. `minmax` works with both fixed and flexible size.
`repeat` allows us to define the same length multiple times.
Lastly, `auto` will set the width automatically to fit the size of the content.
